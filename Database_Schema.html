Database Overview

The Inkspire Library Management System database consists of 7 tables that store information about users, books, categories, borrowing transactions, reviews, favorites, and notifications. The schema follows a relational database design with appropriate foreign key relationships between tables.

Table Relationships

book to category: Many-to-One (A book belongs to one category, a category can have many books)
borrow_request to users: Many-to-One (A user can have many borrow requests, each borrow request belongs to one user)
borrow_request to book: Many-to-One (A book can have many borrow requests, each borrow request is for one book)
review to users: Many-to-One (A user can write many reviews, each review is written by one user)
review to book: Many-to-One (A book can have many reviews, each review is for one book)
favorite to users: Many-to-One (A user can have many favorite books, each favorite entry belongs to one user)
favorite to book: Many-to-One (A book can be favorited by many users, each favorite entry is for one book)
notification to users: Many-to-One (A user can have many notifications, each notification is for one user)
notification to book: Many-to-One (A notification can be related to a book, optional relationship)
